### 4. Server-Side Auto-Generation
**File:** `Controllers/MembersController.cs` - `CreateMember()` method

```csharp
// Auto-generate member number if not provided
if (member.MemberNumber == null || member.MemberNumber == 0)
{
    // Find the highest member number and add 1
    var maxMemberNumber = await _context.Members
        .Where(m => m.MemberNumber.HasValue)
        .MaxAsync(m => (int?)m.MemberNumber) ?? 0;
    member.MemberNumber = maxMemberNumber + 1;
}
else
{
    // Check if member number already exists (only if provided)
    if (await _context.Members.AnyAsync(m => m.MemberNumber == member.MemberNumber))
    {
        return BadRequest(new { error = $"Member number '{member.MemberNumber}' is already in use." });
    }
}
```

**Auto-Generation Logic:**
- Finds the highest existing member number
- Adds 1 to create next number
- Example: If highest is 150 ? assigns 151
- If no members exist ? assigns 1

### 7. Sorting Logic
**File:** `wwwroot/app.js` - `sortMembers()` function

```javascript
// Sort by member number (nulls go to end)
case 'memberNumber-asc':
    sortedMembers.sort((a, b) => {
        const aNum = a.memberNumber || 0;
        const bNum = b.memberNumber || 0;
        if (aNum === 0 && bNum === 0) return 0;
        if (aNum === 0) return 1; // Null values at end
        if (bNum === 0) return -1;
        return aNum - bNum;
    });
```

### SQL Changes Applied
```sql
-- Drop existing unique index
DROP INDEX [IX_Members_MemberNumber] ON [Members];

-- Make column nullable
ALTER TABLE [Members] ALTER COLUMN [MemberNumber] int NULL;

-- Create filtered unique index (only for non-null values)
CREATE UNIQUE INDEX [IX_Members_MemberNumber] 
ON [Members] ([MemberNumber]) 
WHERE [MemberNumber] IS NOT NULL;
```

### What This Means
- ? Multiple members can have `NULL` member number
- ? Non-null member numbers must be unique
- ? Can mix members with and without numbers
- ? Existing data preserved

## Edge Cases Handled

### Case 1: All Members Without Numbers
- Auto-generation starts at 1
- Each new member gets next number
- Works correctly

### Case 2: Mix of Numbered and Unnumbered
- Auto-generation finds max of numbered members
- Unnumbered members display "No #"
- Sorting works correctly

### Case 3: Gaps in Numbering
- If numbers are 1, 3, 5 (missing 2, 4)
- Auto-generation assigns 6 (next after max)
- No gap filling (by design)

### Case 4: Large Numbers
- If max number is 9999
- Auto-generation assigns 10000
- No overflow issues (int can go to 2 billion+)
